## The test for stacking
Does our existing tender build *naturally* satisfy the track, or would we bolt something on for eligibility? Every rubric punishes bolt-ons, and we don't have the hours. Only stack where our core already qualifies.

## Primary — Conduct "Make Legacy Move" (£8k, top 3)
Unchanged. Biggest pot, judge-validated (Henry), fits the team. Everything in the master plan points here.

## Stack candidate — Fetch.ai (£500 / £350 / £150 + internship interviews) ✅ *the only clean stack*
**Why it stacks:** our extraction engine already satisfies the track. Register a "TenderAnalyzer" agent on Agentverse that, given a tender, returns the extracted requirements with mandatory ones flagged — usable inside ASI:One chat. Reuses the core directly; only new work is wrapping it in the uAgents Chat Protocol + registering it (~half-day for J/generalist). **No custom frontend needed** (the agent replies in chat), so it doesn't fight our Conduct controlled-UI build — it's a parallel exposure of the same engine.

**Real cost:** small cash; separate Devpost submission + its own short demo video. Not free.
**Real draw:** the **internship interviews** (matter for J's path) + a credibility line ("our core also runs as a discoverable Agentverse agent").

**Decision rule — revisit Day 3:** do it *only* if the extraction core is solid by ~Day 3 and J/generalist have genuine slack. J owns it as a parallel task; it must **never** pull the engineers off the Conduct build. If the build needs all hands, drop it without a second thought.
**Before committing:** confirm Fetch's submission deadline + that the separate demo video is genuinely a half-day, not a rabbit hole.

## Ruled out (so they don't tempt us)

**Bittensor / Apex — skip.** Not a product track — it's Kaggle-on-mainnet: write ML solutions to win live algorithmic competitions against professional miners. Orthogonal to the tender tool, separate skill set + setup. Nothing to stack.

**Kaspa ($1k) — skip.** Crypto-native: Kaspa as a programmable payments/coordination/settlement layer, scored 30% on Kaspa being *fundamental, not bolted on*. Our single-user document tool has no payment/escrow/settlement element — qualifying means stapling on an on-chain layer we don't need. Exactly the "added for eligibility" pattern the rubric punishes.

**Coral (Solana) — skip.** Same logic as Kaspa — agent-economy with on-chain escrow. No natural hook in our build.

**BasedAI — skip building, attend the workshop.** The track (permission-aware shared-memory layer: ACL sync under concurrency, deterministic sub-200ms-P99 permission checks, lineage-based revocation) is a serious distributed-systems project, not an extension of our tool — the "audit/lineage" overlap with our decision-capture is superficial. Building it = a second full project. **Exception:** prize is BasedAPIs credits + Hirebase + a fast-track interview, with workshops Mon 29 (3–4pm) / Tue 30 (4–5pm). If J has a spare hour, attend purely for the network — zero build commitment. Don't build for the track.
